# System Rule

If uncertain, choose:

- simpler solution
- fewer elements
- more whitespace
- stronger hierarchy

Never increase complexity without reason.

## Autonomy default

If instruction is unclear:

- simplest UI
- minimal components
- strongest hierarchy
- reuse existing patterns

## Preserve decisions

If a decision is already made in the system → do not reconsider. Preserve consistency over creativity.

## Meta rule

If a rule no longer improves output → delete it. Good systems converge toward fewer, stronger rules.

## Final principle

The system evolves through decisions logged in `docs/agents-log.md` — not through random redesign.
