# Everest Design Taste

## Stakeholder preferences (read `docs/design-taste.md`)

- **Clean and presentable** — corporate global trading house, not startup landing.
- **No visual garbage** — avoid busy backgrounds, excessive gradients, decorative noise.
- **No muddy/darkened colors** — keep contrast crisp; brand navy `#0B1F3B` not black mush.
- Inspiration sources (patterns only, not 1:1 copy):
  - https://www.awwwards.com/
  - https://21.dev/

## Workflow

1. Before proposing UI: check `docs/design-taste.md` and `docs/design-bundles.md`.
2. Offer **2–3 bundles** each time major visual decisions are needed:
   - Font pair + Google Fonts link
   - Color palette (primary, accent, text, background)
   - Button style note (radius, hover, shadow level)
3. Default bundle = customer brief (Montserrat, Roboto, `#0B1F3B`, `#C8A45A`).
4. After user picks or dictates preference → update `docs/design-taste.md` § Decisions.
5. Use `ui-ux-pro-max` skill for reasoning: **Trust & Authority** + **Swiss Modernism 2.0**.

## Anti-patterns

- Neon, AI purple/pink gradients, glassmorphism overload, parallax circus
- Emoji as icons
- Too many font families (>2 + CJK fallback)
- Hero with Russia→China arrows or dominant single-route map

## Prototyping

When user asks for prototypes: describe or sketch 2 layout variants in `docs/design-bundles.md` with section order unchanged (8 blocks per brief).
