# Typography System

Typography creates hierarchy before color or decoration.

## Principles

Study patterns from Linear, Vercel, Stripe, Notion, Raycast — extract rhythm, not fonts.

Prioritize: hierarchy · readability · whitespace · rhythm.

A great layout with excellent typography beats a layout with visual effects.

## Everest active fonts

| Role | Font | Files |
|------|------|-------|
| Body | Switzer / DM Sans | `everest-wibify-ui.css`, `everest-site-v2.css` |
| Display emphasis | Instrument Serif | Hero accents, block titles |
| Mono labels | JetBrains Mono | Eyebrows, tags, `[01]` labels |
| CHI | Noto Sans SC | `everest-i18n.js`, embed-head |

## Hierarchy

| Level | Treatment |
|-------|-----------|
| H1 / Hero | Large, whipped lines, serif accent spans |
| Section title | Display + blue/gold accent alternation |
| Subtitle | Muted, smaller, generous line-height |
| Body | Readable, balanced line length |
| Eyebrow | Mono, `[NN]` + label, muted line |

## Rules

- Few font sizes — no random scaling
- Headings: bold, clear, high contrast
- Avoid tiny text and excessive font weights
- Headings feel intentional
- Use spacing + typography to guide attention

## Before creating a page

Determine: primary message · secondary message · visual hierarchy. Build typography around hierarchy first.

## Avoid

- 5+ font sizes in one section
- Inconsistent heading scales
- Cramped line spacing
- Decorative typography without meaning

## Content first

Content drives design. Never create UI without knowing who reads, what they need, what action they take.

## Copywriting

Avoid generic SaaS copy ("Welcome to our platform", "Innovative AI solution").

Prefer: specific · outcome-oriented · measurable benefits. Every heading communicates value.
