# Design Brain

Do not start from components. Start from meaning.

## Design thinking order

1. Define user goal
2. Define hierarchy
3. Define content
4. Define layout
5. Map to components (last)

## UX rules

- User scans, does not read
- Clarity within 3 seconds
- Reduce cognitive load
- Fewer choices = better UX
- If user must think → simplify UI

## Primary user flow

Each page/section: **one primary action**. Everything else is secondary.

## Design priority order

1. Clarity
2. Usability
3. Consistency
4. Aesthetics
5. Animation

## Product logic

Every UI element must answer: **What does the user gain from this?**

If an element does not increase understanding, conversion, or reduce friction → remove it.

## Information architecture

Prioritize: clarity · hierarchy · user journey.

Remove sections that do not support the primary goal (single-page: RFQ funnel).

## Whitespace

Prefer more whitespace. Whitespace is a design element. Breathing room improves perceived quality.

## Product consistency

Every section must feel like the same product. Maintain spacing, typography, color, interaction patterns, tone of voice.

Do not introduce new patterns without reason.

## References

Before designing, analyze Linear, Vercel, Stripe, Raycast — extract **principles**, do not copy layouts.

## Variants rule

Do not generate multiple stylistic variants. Pick one direction and refine it.

## No invention rule

Do not invent new UI patterns, spacing systems, or icon styles. Reuse existing system.

## Critical review mode

Assume: user is distracted · user scans · user wants fastest path.

If UI requires thinking → simplify.

## Product check

Before finishing:

- Is this obvious in 3 seconds?
- Is the next action clear?
- Can the user make a mistake here?

## Product state

This is a real product. Every change affects trust, conversion, perception of quality.

## Art direction

If design feels generic, template-like, or cluttered → iterate again.

Target: premium B2B — not generic startup templates.
